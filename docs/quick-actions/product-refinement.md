Review the presented item against the  [[Product Refinement - Definition of Ready]].
If no item is presented, ask the user to provide one.

Review the item from different angles:

- Product Manager
- Requirements Engineer
- Delivery Manager
- UX Researcher
- Software Architect

Dispatch dedicated Subagents if possible.

Analyze:

1. Customer Problem
2. Cost of the Problem
3. Business Impact
4. Customer Impact
5. Success Metrics
6. Outcome Definition
7. Assumptions
8. Risks
9. Alternative Solutions
10. Scope Reduction Opportunities
11. Missing Information
12. Readiness Assessment

Provide:

### Executive Summary

- Ready / Not Ready
- Readiness Score (0-100)

### Findings

For each category:

- Assessment
- Evidence
- Risks
- Recommendations

### Outcome Statement

Create a concise outcome statement.

### Success Metrics

Suggest measurable KPIs.

### Refinement Backlog

Generate:

- Open Questions
- Discovery Tasks
- Research Tasks
- Refinement Tasks

### Execution Plan

Generate:

1. Discovery Phase
2. Validation Phase
3. Design Phase
4. Refinement Phase
5. Delivery Phase

### Recommendation

Choose one:

- Proceed
- Refine
- Research Further
- Reject

```text
<INSERT PRODUCT OR FEATURE HERE>
```
